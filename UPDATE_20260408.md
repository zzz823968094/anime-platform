## 常见问题

### Q1: 构建失败，提示找不到 JAR 文件

**解决方案**：确保 Maven 编译成功

```bash
# 手动编译
cd /opt/anime-platform
mvn clean package -DskipTests

# 检查 JAR 文件是否生成
ls -lh gateway/target/*.jar
ls -lh anime-service/target/*.jar
ls -lh user-service/target/*.jar
```

### Q2: 服务启动后立即退出

**查看日志**：

```bash
# 查看具体错误
docker-compose logs [服务名]

# 例如
docker-compose logs gateway
docker-compose logs anime-service
```

**常见原因**：
- 数据库连接失败：检查 MySQL 是否完全启动
- Nacos 连接失败：检查 Nacos 是否完全启动
- 端口冲突：检查端口是否被占用

### Q3: 服务启动慢

**正常现象**：Spring Boot 应用启动需要 30-60 秒

**查看启动进度**：

```bash
docker-compose logs -f gateway
```

等待看到类似 "Started GatewayApplication in xx seconds" 的日志。
