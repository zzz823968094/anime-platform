# 动漫平台 (Anime Platform)

一个基于 Spring Cloud 微服务架构的动漫内容综合平台，支持动漫浏览、用户管理、视频播放、实时弹幕互动等功能。

## 项目简介

本项目是一个采用微服务架构的动漫内容平台，采用 Spring Cloud 作为微服务框架，Nacos 作为服务注册中心，MySQL 作为数据库，Redis 作为缓存。整个系统包含多个微服务模块，共同构成完整的动漫平台功能。

## 技术栈

后端核心技术采用 Java 17 和 Spring Boot 框架，具体包括 Spring Cloud Gateway 作为 API 网关、Spring Cloud OpenFeign 实现服务间调用、MyBatis Plus 作为 ORM 框架。数据存储方面使用 MySQL 作为主数据库，Redis 作为缓存和会话存储。服务注册与配置中心采用 Alibaba Nacos。部署方面使用 Docker 容器化技术和 docker-compose 进行服务编排。

## 系统架构

项目采用多服务微服务架构，各服务模块职责分明。gateway 模块作为系统的统一入口，提供 API 网关功能，负责请求路由、身份认证和限流控制。anime-service 模块是动漫内容管理核心服务，提供动漫信息 CRUD、搜索、推荐等功能。user-service 模块负责用户管理，包含用户注册、登录、收藏等用户相关功能。video-service 模块管理视频内容，提供视频播放地址和集数管理。crawler-service 模块是数据爬取服务，自动从外部获取动漫数据并同步到数据库。danmaku-service 模块提供实时弹幕功能，通过 WebSocket 协议实现实时消息推送。

## 核心功能

动漫内容管理模块支持多种维度的动漫检索，包括按类型、状态、年份、风格分类筛选，支持按最新更新、热度和评分进行排序推荐，支持关键词搜索和搜索历史统计。该模块还提供热门推荐和最新更新的自动推荐功能，以及访问统计和用户行为记录。

用户管理模块提供完整的用户账户功能，包括用户注册、登录验证（支持 JWT Token）、收藏管理（添加、删除、查询、检查收藏状态）。系统还实现了注册限流防护，同一 IP 每天最大注册次数有限制。

视频播放模块支持根据动漫 ID 获取该动漫的所有剧集列表，支持获取单个视频详情，提供播放地址的动态路由。

弹幕互动模块采用 WebSocket 协议实现真正的实时弹幕功能，支持按视频 ID 加入房间、离开房间、广播消息到指定房间，并自动将弹幕内容持久化到数据库。

数据爬虫模块实现自动化数据同步，支持增量同步和全量同步，支持按类型（日本、美国、中国动漫）分别爬取，支持失败重试和缺失封面的自动补充填充。爬虫任务支持定时执行，日本动漫每 3 小时同步一次，其他地区每 6 小时同步一次。

## 快速开始

### 环境要求

部署环境需要 Java 17 或更高版本、Docker 和 docker-compose、Linux Ubuntu 系统（推荐 Ubuntu 20.04 或 22.04）。

### 部署步骤

首先克隆项目代码到服务器：

```bash
git clone https://gitee.com/crazy-clown/anime-platform.git
cd anime-platform
```

然后配置环境变量，创建 .env 文件并设置必要的环境变量，包括数据库密码和版本标签。

接着启动所有服务：

```bash
./deploy/deploy.sh
```

首次部署需要构建镜像，可能需要较长时间。服务启动后可以通过 docker ps 查看运行状态。

### 服务端口

系统各服务的端口分配如下：Gateway 服务监听 8080 端口，anime-service 服务监听 8081 端口，user-service 服务监听 8082 端口，video-service 服务监听 8083 端口，danmaku-service 服务监听 8084 端口，crawler-service 服务监听 8085 端口，MySQL 数据库监听 3306 端口，Redis 缓存监听 6379 端口，Nacos 服务注册与配置中心监听 8848 端口。

## API 接口

平台提供丰富的 RESTful API 接口。动漫相关接口包括获取动漫列表（支持分页和多种筛选条件）、获取动漫详情、删除动漫、上线动漫、搜索动漫、获取搜索统计、获取热门推荐、获取最新更新、获取统计信息、获取最近搜索记录、报告访问记录等。用户相关接口包括用户注册、用户登录、获取用户信息、获取当前用户详情、获取用户列表、获取用户数量统计、添加收藏、取消收藏、获取收藏列表、检查是否收藏等。视频相关接口包括根据动漫 ID 获取视频列表、获取单个视频详情等。爬虫相关接口包括立即爬取、全量同步、增量同步、按类型同步、重试失败、获取失败数量、填充封面缺失等。弹幕相关接口通过 WebSocket 协议连接，连接地址为 ws://host:8084/danmaku，支持 join（加入房间）、leave（离开房间）、send（发送弹幕）等消息类型。

## 项目结构

项目采用了 Maven 多模块结构，根 pom.xml 定义了所有子模块和依赖版本。gateway 模块是 API 网关服务。anime-service 模块是动漫内容管理服务。user-service 模块是用户管理服务。video-service 模块是视频内容服务。crawler-service 模块是数据爬虫服务。danmaku-service 模块是弹幕互动服务。common 模块是公共工具和基础类定义。deploy 目录包含 Docker 部署相关文件和脚本。

## 注意事项

生产环境部署时建议修改默认密码，特别是数据库 root 密码和 JWT 密钥。只开放必要端口，避免不必要的安全风险。可以根据服务器配置调整 JVM 参数优化性能。定期备份数据库数据，建议使用 cron 定时任务进行自动化备份。系统默认配置适合开发测试环境，生产环境请根据实际需求调整配置。
